## 为什么需要 AI 开店选址专家？

对于实体创业者、加盟商和投资人来说，选址是决定生死的关键一步。但传统的选址调研往往极其低效且充满陷阱：

- 💸 **“花了几万块买选址报告”** → 找咨询公司，收费高昂，等半个月才出一堆用不上的宏观数据。
- 盲 **“我觉得这里人流量挺大”** → 凭直觉去盘算，不知道旁边的地铁口全是匆忙的“假客流”。
- ☕ **“准备 80 万在杭州开个咖啡馆”** → 到底在哪个区、哪个板块、选择什么模式最不容易死？没有标准答案。
- 🗺️ **“去地图上搜一下竞品数”** → 自己手动查，根本没法建立消费水平、租售比预警、存量红海指数的综合三维模型。

**这些分析不是大模型做不到，而是大模型没有“商业思维逻辑”和“真实的地理数据”。**

普通的 AI 只能给你一句：“建议您综合考虑人流量和租金。”这完全是废话。

**AI Location Scout 把选址变成了一场专业的沉浸式数字推演：**

```
帮我安装这款牛逼的 AI 开店选址专家，仓库地址是：https://github.com/yyuchenBaii/ai_location_scout，装好后告诉我。
```

把这句话发给你的 Agent（比如 OpenClaw），几秒钟后，它直接变身为具有 10 年经验的商业地产合伙人。

它不仅会反问你**极其尖锐的商业模式问题**，还会**后台挂载真实的高德 LBS 数据抓取工具**，最终一键吐出一张包含 10 维雷达图、极度暗黑科技感、且可点选交互的**《选址多维评测网页大屏》**！

## 快速上手

### 🔌 方式一：终端极速对话安装 (如果你使用 OpenClaw)

不需要手动下载，不需要找文件夹配置。直接复制这句话给小龙虾：

```
帮我安装从Github上安装这个选址 Skill，仓库地址是：https://github.com/yyuchenBaii/ai_location_scout，装好后告诉我。
```

小龙虾会在后台自动拉取大脑配置（`SKILL.md`）、网页骨架（`resources/*`）和数据爬虫（`scripts/*`），并自动存放到 `.agents/skills` 目录。
装完后，直接提问“帮我评估一下某个地址”，立刻见证奇迹。

### ⚙️ 方式二：常规平台安装 (如 Coze / Dify 等)

1. **装载大脑**: 将 `SKILL.md` 的内容全文复制，填入你机器人的“提示词 / 人设角色”框中。
2. **连接高德数据源 (必做)**:
   - 前往 [高德开放平台](https://console.amap.com/) 申请一个免费的 **Web 服务 API Key** 和一个 **Web 前端 JS API Key**。
   - 将 Web 服务 Key 填入 `scripts/fetch_amap_poi.py`，并将该 Python 脚本作为“工具 / 插件”挂载给大模型，让它拥有真正的“眼睛”。
   - 将 JS API Key 填入 `resources/` 下的所有 HTML 模板头部，这样 Agent 返回给你的研报里的地图就能成功渲染了。

<details>
<summary>它到底是如何工作的？（点击展开原理）</summary>

1. **意图路由引擎**：Agent 读取你的提问，走查 8 种核心场景，决定该问你什么，该用哪种逻辑帮你算账。
2. **外部真实工具调用（Function Calling）**：当你确认了要开什么店、在哪个经纬度后，Agent 会暂停瞎想，后台调用 `fetch_amap_poi.py` 获取真实竞品密度和最近的直接死磕对手。
3. **10 层深度数据切片模型**：Agent 拿到真实数据后，结合大模型常识，开始穿透评估（宏观覆盖、潮汐客流、真实转化、租售比红线）。
4. **前端沉浸式降频交付**：Agent 将推演出的百项数据指标，揉捏进我们预设好的 `resources/` 的高维暗黑大屏 HTML 模板里，然后在你的屏幕上呈现互动视图。

## 核心设计理念

**“选址不是为了找到流量最大的地方，而是为了找到死的最慢的地方。”**

本系统不是一个找地图坐标的普通插件，它遵循以下三大核心理念：

### 1. 拒绝冰冷数据堆叠，提供“洞察与刺痛”
市面上的报告只会告诉你“咖啡店存量多”。而本智能体会在最后生成一条**极度洞察的一句话防坑指南**：
> *"这里距离最近的瑞幸仅 50 米，且园区白领自带大厂补贴免费咖啡闭环。以你的资金储备进入此局，三个月现金流极大概率断裂。千万别碰！"*

### 2. 真数据托底，杜绝 AI 幻觉
所有的红海指数、存留几率不是瞎编的。它是通过强制调用本地 `scripts/` 下的 Python 插件连线高德，看一眼真实世界到底有多残酷，再拿回来给大模型当燃料。

### 3. All-in-One 沉浸体验 (前端级封装)
最终发给用户的不是 Markdown 写的毫无重点的烂列表，而是一段带 JS 和 CSS 的精致 HTML 页面。用户就在原地的聊天框内，看雷达图扭转，看地理大屏刷新。体验降维打击传统的报表输出。
